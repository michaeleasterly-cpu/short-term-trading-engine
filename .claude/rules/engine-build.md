# Engine-build compliance shortlist

Canonical SoT: `docs/superpowers/checklists/engine_readiness.md` (the 10-section non-optional gate the SDLC planner machine-checks via `planner._check_readiness`).
Authoritative external: <https://code.claude.com/docs/en/extend> (rules), <https://code.claude.com/docs/en/memory> (CLAUDE.md as project memory).

Every engine plug under `<engine>/plugs/` and the template must satisfy these on every touch:

- **Every plug** subclasses `tpcore.interfaces.engine_plug.BaseEnginePlug` and implements `validate_dependencies` + `healthcheck`.
- **`setup_detection` populates a `tpcore.backtest.filter_diagnostics.FilterDiagnostics`** so SIGNAL events carry per-gate pass/block counters.
- **Backtest calls `write_credibility_score`** so the capital gate has a rubric row to read. Exception: `canary` is intentionally non-graduating (spec §4b) and MUST NEVER call it.
- **Scheduler checks `tpcore.calendar.is_trading_day()`** and returns early on non-trading days.
- **AAR uses `tpcore.aar.classify_exit_reason`** — never hardcode `ExitReason` literals.
- **Stale-order cancel via the shared `tpcore.order_management.stale_order_cancel`** (P5.4); momentum/sentinel keep a 1-line delegate.
- **Scheduler `await db_log.startup()`** after the `try:` opens and **`await db_log.shutdown(duration_ms=..., exit_code=...)`** in `finally:` — without these the dashboard cannot see the run.
- **Smoke-loop registration** (the sentinel-fenced region regenerated by `python scripts/gen_engine_manifest.py`): added at build time, not after. `gen_engine_manifest.py --check` reds CI on drift. Do NOT hand-edit inside a fence.
- **`run_all_engines.sh` + its docstring listing + `ops/platform_pipeline.py` docstring listing** are sentinel-fenced; same generator.
- **Register every required ticker** (regime gate, basket member, market-context proxy) in `CRITICAL_TICKERS` in `tpcore/quality/validation/checks/prices_daily_freshness.py` — the SPY-gap incident (2026-05-15) is why.

Roster + lifecycle changes go through the ECR ONLY (see `engine-roster` rule) — never hand-edit `tpcore.engine_profile._PROFILE`.

Start from `tpcore/templates/engine_template/`; the scaffold satisfies the gaps by construction.
