STANDING RULE: Public dashboard pages (/southern-illinois, /carbondale, /murphysboro, /market, and any future public pages serving real stakeholders) must never lead with editorial / judgment language in headlines, eyebrows, hero tags, or section titles.

**Banned in headlines:** "structurally weak", "hollow", "fine vs. hollow", "phantom", "broken", "failing", "theater", "in-trouble", and any adjective that asserts a verdict.

**Allowed in headlines:** region/timestamp/data-source identifiers ("LWA-25 · BLS LAUS · 2026-03"), raw numbers with comparisons ("UR 5.3% · LFPR 56.7% (−8.4pp vs IL)"), category names ("Labor market", "Federal awards"), and the source-citation suffixes already standard on the page.

**Where judgment language is fine:** Inline analysis paragraphs that name a specific source ("the 8.4pp gap means 75,950 working-age people have stopped looking — they are not in the headline UR" — facts, sourced) and operator-validated callout blocks. But never as the dominant H1/eyebrow framing.

**Why:** Operator (2026-05-27) on the SI scaffold example: "structurally weak labor market should not be the fucking headline. Right? Like, that's a fucking judgment." The whole point of the page is that the workforce-development theater problem is visible IN THE NUMBERS — if my headline pre-labels the conclusion, I'm doing the same theater the page is supposed to expose. Reader trust on these pages depends on appearing data-driven, not opinionated. Same lens as [[feedback_research_builder_persona]] and [[feedback_no_lazy_vendor_blame]] — let the data carry the argument.

**How to apply:** Before publishing any new hero/eyebrow/section-head copy on a public dashboard page, ask: does this sentence contain a verdict word, or only data + identifier + neutral category? If verdict word → rewrite to data-first. The design scaffold's example content (Southern Illinois Dashboard.html) uses editorial framings for *demonstration* — those examples do not transfer to the live page. Apply the scaffold's typography + colors + component classes, not its example copy.
